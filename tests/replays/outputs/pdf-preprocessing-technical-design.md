# PDF 预处理架构技术设计

## 背景与问题

当前系统接收三类文件：文本型 PDF、扫描型 PDF、混合型 PDF。下游检索
和问答模块需要稳定的页级文本、图片、元数据和错误状态。预处理层的任务
不是提高模型效果本身，而是把输入差异收敛成可检查的数据记录。

## 目标与非目标

目标：

- 识别文件级和页级类型。
- 抽取文本、页面图片、基础元数据和处理状态。
- 记录可重试错误，避免静默丢页。
- 为索引层提供统一的页级输出。

非目标：

- 不做 PDF 编辑、合并、拆分或表单填写。
- 不承诺 OCR 准确率，准确率需要样本集验证。
- 不在预处理层完成业务语义抽取。

## 输入与输出

| 项目 | 内容 |
| --- | --- |
| 输入 | PDF 文件、来源、任务 id、处理配置。 |
| 输出 | 页文本、页图片、页元数据、错误记录、处理耗时。 |
| 状态 | `queued`, `parsed`, `ocr_needed`, `failed`, `done`。 |

## 架构方案

预处理层分为五个组件：

- 接入器：校验文件大小、页数和 MIME 类型。
- 类型识别器：判断文件和页面是否需要 OCR。
- 抽取器：对文本页执行直接抽取。
- OCR 调度器：把扫描页送入 OCR 队列。
- 结果归一化器：写入统一页级记录。

## 处理流程

1. 接入器创建任务记录并保存原始文件引用。
2. 类型识别器读取页面结构，标记文本页和扫描页。
3. 抽取器处理文本页，OCR 调度器处理扫描页。
4. 归一化器合并页文本、图片、元数据和错误状态。
5. 校验器检查页数一致性、空文本比例和失败页列表。

## 错误处理

错误记录需要包含任务 id、页码、组件、错误码、重试次数和原始异常摘要。
可重试错误进入队列，不可重试错误保留在页级输出中，供人工或下游模块
决定是否跳过。

## 性能与扩展性

文本页可以同步抽取。扫描页需要进入 OCR 队列，队列并发上限由 OCR 服务
吞吐、平均页数和成本预算决定。批量任务应记录每个组件的耗时，避免只看
总耗时而无法定位瓶颈。

## 验证计划

- 使用 100 份混合 PDF 样本验证页数一致性。
- 对文本页统计空文本比例。
- 对扫描页记录 OCR 成功率和平均耗时。
- 随机抽查失败页，确认错误码可解释。

## 风险与下一步

主要风险是 OCR 成本、异常 PDF 格式和页级状态不一致。下一步应先确定
样本集、错误码表和输出 schema，再接入真实批量任务。
