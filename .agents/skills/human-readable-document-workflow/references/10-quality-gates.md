# Quality Gates

Before final delivery, check the document against this list.

## Goal And Reader

- Does the document match the user's stated goal?
- Is the target reader clear?
- Does the structure answer the reader's likely questions?

## Style

- Is obvious AI phrasing removed?
- Are claims concrete and supported?
- Are endings useful instead of preachy?
- Are lists used only when they improve scanning?

## Facts

- Does the draft avoid invented facts, sources, citations, metrics, or quotes?
- Are assumptions labeled?
- Are missing sources marked with `citation needed` or a user-facing note?

## Structure

- Is the heading hierarchy clear?
- Is Markdown renderable?
- Are code blocks, tables, figures, and formulas formatted consistently?

## Output Path

- Is the target format consistent with the user's request?
- If Word/PDF is requested, is the Markdown source available?
- Is the renderer path clear?
- Are limitations or missing tools stated plainly?
