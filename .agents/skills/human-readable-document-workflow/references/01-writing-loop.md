# Writing Loop

Use a loop instead of jumping from prompt to final draft.

## Flow

1. Intake: collect goal, reader, use case, source material, format, length, tone, must-keep content, and forbidden changes.
2. Frame: define the problem, reader questions, and delivery boundary.
3. Plan: create a short structure with stable headings.
4. Draft: write the first coherent version.
5. Critique: check logic, factual boundaries, style, structure, and renderability.
6. Revise: improve clarity, continuity, and usefulness.
7. Finalize: normalize the source and provide the requested output or render path.

## Rules

- Do not skip directly to a polished-looking but shallow final answer.
- Keep the visible process proportional to the task; do not expose internal critique unless the user asks.
- When learning from examples, abstract style only from user-approved good documents.
- Do not learn habits from weak AI drafts, including padded introductions, empty endings, and mechanical transitions.
- If source material is incomplete, mark gaps instead of inventing facts.
