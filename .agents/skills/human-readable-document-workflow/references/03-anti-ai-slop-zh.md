# 中文去 AI 腔规则

目标是减少模板化、说教感和空泛拔高，不是把文字改得口语化或故意粗糙。

## 高频模式

| 问题 | 表现 | 替代策略 |
| --- | --- | --- |
| 宏大开头 | 在当今时代、随着社会发展 | 直接交代具体任务或背景 |
| 套话总结 | 总而言之、综上所述具有重要意义 | 收束到行动、限制或下一步 |
| 机械转折 | 首先、其次、最后密集堆叠 | 用真实逻辑关系连接段落 |
| 伪深刻 | 不仅是工具，更是理念 | 说清具体机制和影响 |
| 过度排比 | 提升效率、优化体验、赋能发展 | 保留可验证的收益 |
| 营销腔 | 极致、领先、革命性、全方位 | 换成可观察的事实 |
| 万能意义句 | 对行业发展具有深远意义 | 说明对谁、在什么条件下有用 |
| 空泛形容词 | 高效、智能、完善、强大 | 补充指标、场景或例子 |
| 无来源判断 | 显然、众所周知、业界普遍认为 | 给来源，或改为假设 |

## 坏例子与好例子

坏：随着数字化转型的不断深入，PDF 预处理架构具有重要意义，能够全方位提升系统效率。

好：PDF 预处理架构要解决三个具体问题：文本抽取不稳定、扫描件 OCR 成本高、后续索引需要统一字段。

坏：总而言之，该方案不仅提升效率，也为未来发展奠定坚实基础。

好：如果后续要支持批量 OCR，应先补齐任务队列和失败重试，否则吞吐会被单文件处理拖慢。

坏：首先我们要分析需求，其次设计架构，最后完成落地。

好：需求约束决定架构边界：输入格式、并发规模和错误恢复方式需要先定下来。
