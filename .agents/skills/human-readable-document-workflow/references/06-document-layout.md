# Document Layout

Layout should help deterministic renderers produce stable output.

## General Rules

- Start Markdown documents with one H1.
- Do not skip heading levels.
- Keep paragraphs moderate in length.
- Use tables only for real comparison or structured data.
- Mark code blocks with a language name.
- Give figures, tables, and formulas explanatory captions or lead-in text.
- Avoid manual alignment with spaces.
- Avoid layout that depends on invisible whitespace.

## Word And PDF

- Treat Markdown as the source of truth.
- Use templates, reference documents, or renderer options for page style.
- Do not simulate Word/PDF layout by inserting repeated spaces or blank lines.
- Prefer simple, stable structures over complex nested tables.
