# Style Distillation

Use this reference only when the user asks to learn, match, imitate, continue,
or reuse a document style from a sample. Do not load it for ordinary drafting,
polishing, or anti-slop cleanup.

## Scope

Style distillation creates a session-only style pack unless the user explicitly
asks for durable storage. It extracts reusable writing behavior, not content
facts from the sample.

Use when the user says things like:

- "write like this sample"
- "match this voice"
- "learn this document style"
- "keep the same report tone"
- "continue in this style"

Do not use when the sample is only a factual source, citation source, data
source, or temporary example of formatting.

## Style Pack Versus Preference

| Item | Meaning | Lifetime |
| --- | --- | --- |
| Style pack | A reusable description of how a sample writes. | Session-only by default. |
| Preference | A user decision about how future work should be written. | Only durable if explicitly stated. |

Examples:

- Style pack: "short paragraphs, concrete nouns, restrained transitions."
- Preference: "For future README work, avoid marketing language."

Do not turn a one-time sample into a durable user preference unless the user
clearly asks for that.

## Style Versus Content Boundary

Extract only reusable form:

- tone and register
- sentence rhythm
- paragraph shape
- abstraction level
- argument structure
- transition style
- terminology habits
- heading habits
- reader relationship

Do not extract:

- facts or events
- names of people, clients, companies, or products
- private business information
- quotes or distinctive source passages
- citations or bibliography entries
- plot facts or examples tied to the sample
- one-time constraints from the current task

If the target document needs facts, use only the facts supplied for the target
document. Never import facts from the style sample.

## Confidence Rules

| Evidence | Confidence |
| --- | --- |
| One short sample | Low. Use broad style hints only. |
| One complete document | Medium. Extract document-level patterns. |
| Multiple approved samples | High. Extract recurring patterns. |
| User confirms a style summary | High for confirmed traits only. |

When confidence is low, state that the match is approximate or ask for another
sample if the result depends on precision.

## Style Pack Shape

Use a compact internal style pack:

```markdown
Style pack:
- Register:
- Sentence rhythm:
- Paragraph shape:
- Structure:
- Transitions:
- Terminology:
- Headings:
- Reader relationship:
- Avoid:
- Confidence:
```

Keep the pack descriptive. Do not include copied source sentences unless the
user explicitly asks for quoted comparison.

## Style Match Review

After applying a style pack, review the draft for:

- style traits that were missed
- style traits that were over-applied
- content facts accidentally imported from the sample
- protected terms, citations, code, or data changed for style
- mismatch between document type and borrowed voice

If the borrowed style conflicts with the target document type, document type
wins. Academic, technical, legal, medical, financial, and policy-like documents
must keep precision over voice mimicry.

## Missing Style Pack Handling

If the user asks to "use the same style" but no sample or prior approved style
pack is available:

1. Say that no reusable style pack is available in this session.
2. Ask for a sample, or proceed with the nearest document profile if the task
   is not blocked.
3. Do not invent a style pack from memory.

## Storage Boundary

Do not write durable style packs, preference journals, or memory files unless
the user explicitly asks for durable storage and names the target location.

Default behavior is session-only: use the style pack for the current task, then
discard it.
