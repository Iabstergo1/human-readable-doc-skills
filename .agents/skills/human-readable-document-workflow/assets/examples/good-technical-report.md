# PDF 预处理架构设计

## 背景

当前系统需要接收文本型 PDF、扫描件和混合 PDF。预处理层的目标是把
这些输入转换成统一的页面文本、图片、元数据和错误记录，供后续索引与
问答模块使用。

## 输入与输出

| Item | Description |
| --- | --- |
| Input | PDF 文件、文件来源、上传时间、用户或任务 id。 |
| Output | 页面文本、页面图片、结构化元数据、错误记录。 |

## 设计要点

预处理流程分为格式识别、页面拆分、文本抽取、OCR、字段标准化和质量
检查。文本型 PDF 优先走直接抽取；扫描页进入 OCR 队列；失败页保留
原始页号、错误类型和重试状态。

## 风险

OCR 成本会随页数线性增长。批量任务需要队列、并发上限和失败重试，
否则单个大文件会阻塞后续文件。
